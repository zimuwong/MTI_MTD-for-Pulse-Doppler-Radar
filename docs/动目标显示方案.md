# 动目标检测方案

## 理论基础

### 脉冲雷达中的多普勒效应

通过对脉冲雷达回波信号进行建模，可以列出这样的数学表达式：

$$
exp(-j\Omega_0 \frac{2d}{c})X(t)
$$

上式是假定目标固定，即目标与雷达之间的距离不变。当考虑目标运动，此时$d=d(t) \approx d_0 + v_0 t$，代入之后观察相位项，可以分离出变化部分：

$$
\exp(-j\Omega_0 \frac{2v_0 t}{c})
=\exp(-j 2\pi \frac{2v_0}{\lambda_0} t)
$$

此式说明回波信号的频率发生了偏移，也即多普勒频移。那么在假设同一脉冲中距离不变的情况下，通过对每一个脉冲采样一个点，相位的变化也就代表了多普勒频率，从而代表着物体移动的速度。

### 脉冲雷达数字信号处理流程

脉冲雷达发射的信号是一组脉冲序列，连续两个脉冲之间的时间间隔称为脉冲重复间隔（PRT），同时也定义脉冲重复频率（PRF）。一组脉冲含有M个脉冲，那么$M*PRT$被称为相干时间。回波信号在被采样后，可以储存在一个数据矩阵中，其中一行代表一个脉冲内的信息（快时间采样），一列代表每一个脉冲相同延时的回波信息（慢时间采样）。

脉冲雷达发射的信号波形一般选择线性调频脉冲，LFM解决了距离探测极限（脉冲持续时间，能量）和距离分辨率（带宽）之间的矛盾。线性调频的发射持续时间很长，频率跨度带宽很大。

脉冲雷达其中比较关键的信号处理流程一般为脉冲压缩-MTD-CFAR-参数解算。

首先对于信号的快时间维进行匹配滤波（又称脉冲压缩）将脉冲变成了短的尖峰（**时域上压缩，频域上匹配**）。理想的匹配滤波器与信号在幅度上完全一致 (最大化信噪比)；相位上完全相反 (相位校正)。处理后，每一行数据中目标的能量会集中在特定的采样点（距离门）上，对应目标的距离。

$$
Y(f) = \underbrace{e^{j\phi(f)}}_{\text{信号}} \cdot \underbrace{e^{-j\phi(f)}}_{
\text{滤波器(共轭)}}
$$

接着对于矩阵的每一列（慢时间维度）进行动目标检测处理，这部分处理一般有两个目的，滤除杂波以及分辨速度。

然后就要对处理完的数据矩阵进行目标判决，最后根据检测到的目标索引提取数据，计算实际物理参数。

### 动目标检测（MTD）

#### 雷达杂波

雷达的工作场景中，除了期望目标之外，还有大量背景噪声，比如雷达波通过地面，楼宇等反射回来的信号。在雷达固定的情况下，杂波的多普勒特性易知频谱上位于零频。

通过数学建模，我们可以根据物理位置，将地面分为一个个patch并在基础上计算RCS从而得到平均回波强度。杂波是由天线波束照射区内大量散射单元回波合成的结果,所以杂波的起伏特性一般符合高斯分布。同时多普勒频率为0。

#### 杂波抑制参数

雷达利用MTI/MTD技术进行杂波抑制的效果主要通过三个指标来量化。首先是**杂波衰减 (CA) 与对消比 (CR)**，这两个指标仅关注杂波本身的被抑制程度。其中杂波衰减定义为滤波器输入端与输出端杂波功率的比值；而对消比则是倒数平方的数学关系。

其次是**改善因子 (I)**，这是衡量滤波器性能最全面的指标，它定义为输出端信杂比与输入端信杂比的比值。 $I = G \times CA$ ，将信号的平均功率增益 $G$ 与杂波衰减 $CA$ 结合起来，意味着理想的滤波器不仅要深度压制杂波，还要尽可能保留各种速度下的目标信号能量。

最后是**杂波可见度 (SCV)**，它代表了雷达在强杂波背景下的实际探测能力极限，即衡量能检测到多微弱的目标。SCV 在数值上等于改善因子减去检测门限（可见度系数 $V_o$，工程上常取 6dB），直观地表示了即便输入端的杂波功率比信号功率大出很多倍（即 SCV 值），雷达依然能保证输出端的信杂比达到检测门限从而发现目标。此参数与实际观测环境关系较大，不可分离成单独指标。

#### MTI杂波抑制滤波器

以地杂波为例,杂波谱通常集中于直流(多普勒频率 fa=0)和PRF的整数倍处。在脉冲雷达中 MTI 滤波器就是利用杂波与运动目标的多普勒频率的差异,使得滤波器的频率响应在杂波谱的位置形成凹口,以抑制杂波,而让动目标回波通过后的损失尽量小或没有损失。

<img title="" src="assets/2026-01-26-17-17-45-image.png" alt="" width="443" data-align="center">

常用的MTI滤波器是延迟线对消器，根据对消次数的不同，分为单延迟线对消器，双延迟线对消器和多延迟对消器。

**单延迟线对消器**，指的是利用一节延迟时间等于雷达脉冲重复周期（$T_r$）的延迟线和一个减法器组成的系统。

其工作原理是将当前接收到的回波信号与前一个重复周期的回波信号进行相减，时域表达式为 $y(t) = x(t) - x(t-T_r)$。对于静止的固定杂波，相邻两个周期的回波幅度与相位基本保持不变，相减后输出为零，从而被消除；而运动目标的回波相位会因多普勒效应随时间变化，相减后会有剩余信号输出。

从频域特性上看，单延迟线对消器的频率响应函数为 $|H(f)| = 2|\sin(\pi f T_r)|$。

<img title="" src="assets/2026-01-26-17-23-56-image.png" alt="" data-align="center" width="283">

**双延迟线对消器**，由两个单延迟线对消器级联而成。

在时域上，输出信号与输入信号的关系为 $y(t) = x(t) - 2x(t-T_r) + x(t-2T_r)$，即它利用当前的脉冲、前一个脉冲和前前一个脉冲（共三个脉冲）加权求和来产生输出，其中的权重系数 "1, -2, 1" 对应二项式展开的系数。

从频域特性上看，由于它是两级级联，其频率响应是单延迟线对消器的平方，幅频特性函数为 $|H(f)| = 4\sin^2(\pi f T_r)$。

<img title="" src="assets/2026-01-26-17-32-36-image.png" alt="" data-align="center" width="462">

由一阶二阶对消器的频率响应可以判断，二阶对消器：零点处更深，对静止杂波抑制更强，通带峰值更高，但过渡更陡；一阶对消器：零点处抑制较弱，通带响应更平缓，对运动目标信号保留更均匀。杂波强时，用二阶对消器（抑制更好），波弱或需要更好保留信号时，用一阶对消器。

<img title="" src="assets/2026-01-26-17-35-12-image.png" alt="" width="487" data-align="center">

#### 动目标检测（MTD）

MTD (Moving Target Detection) 是一种利用**多普勒滤波器组**来抑制各种杂波，以提高雷达在杂波背景下检测运动目标能力的技术。

设计 MTD 滤波器组常见的方法就是 **MTI 级联 FFT 方法**。其结构是先经过一个 MTI 对消器，然后紧接着进行 FFT 运算（用于相干积累和测速）。使用FFT是由于离散傅里叶变换（DFT）本质上就是一组中心频率均匀分布的窄带滤波器组，因此采用FFT算法来高效实现。

## 系统功能完善

### 组间参差

MTI 滤波器本质上是一个离散时间的数字滤波器，其频率响应具有周期性。由于滤波器被设计为在零多普勒频率处产生凹口以抑制静止杂波，这种凹口也会周期性地出现在脉冲重复频率（PRF）的整数倍处。这意味着，当运动目标的径向速度产生的多普勒频率恰好是 PRF 的整数倍时，该目标会被当作静止杂波误切除，即产生所谓的“盲速”。为了解决这一问题，系统必须引入参差 PRF 技术。针对本系统采用的脉冲多普勒体制，考虑到后续 MTD 处理需要对相干处理间隔（CPI）内的数据进行 FFT 运算，这要求同一组内的数据采样间隔必须均匀，因此传统的脉冲间参差（即每个脉冲改变一次 PRF）不再适用，必须采用组间参差方案。

组间参差的具体实施方式是在不同的 CPI 之间切换 PRF 值。例如，在第一个 CPI 使用 PRF1 进行发射和接收，紧接着在下一个 CPI 切换到 PRF2。这种机制会导致不同组生成的数据立方体在时间维度上不一致，因此 MTI 和 MTD 模块需要在各自的 CPI 组内独立进行处理。处理完成后，系统会得到两组基于不同 PRF 的速度测量值。虽然单一 PRF 下的目标速度可能存在模糊，但通过对比两组解算结果，利用中国余数定理或重合检测算法，可以解算出目标的真实速度。这不仅有效填补了单一 PRF 的盲速区域，还将系统的最大可检测不模糊速度扩展到了多个 PRF 的最小公倍数范围，显著提升了系统对高速目标的检测能力。<img title="" src="assets/3836628f9cf1ddb86cd757fe58c5357c4971a466.png" alt="" width="309" data-align="center"><img title="" src="assets/2026-01-28-16-20-20-image.png" alt="" width="295" data-align="inline"><img title="" src="assets/2026-01-28-16-10-23-image.png" alt="" width="290" data-align="inline">

### 杂波图建立

为了实现对动目标的高效检测同时最大限度地保留目标，系统引入了杂波图机制来实现 MTI 的精细化控制。杂波图在工程本质上是一个与雷达距离单元一一对应的存储表，用于记录每个空间位置的历史杂波强度。系统并不依赖瞬时的回波能量来直接控制滤波器，因为单次扫描的数据容易受到噪声或偶然干扰的影响，造成控制抖动。相反，在每个 CPI 期间，系统对各距离门的回波数据进行零频滤波，提取出代表静态地物强度的直流分量。该分量并不直接使用，而是与存储器中的历史杂波强度进行递归加权更新，以平滑随机噪声波动，形成稳定的背景杂波分布图。

在 MTI 处理阶段，模块根据当前距离门在杂波图中的强度值进行实时判决。这种机制不需要复杂的实时计算，而是通过“查表+判决”的方式，实现了滤波器阶数随地理环境的动态切换，在保证强杂波区不虚警的同时，最大程度地保护了弱杂波区的目标能量。

## 系统设计

### 设计要求与方案制定

#### 系统指标分析与设计要求

本系统为 Ku 波段二维相控阵脉冲压缩雷达，主要用于对地面及低空微小目标（RCS $\ge 0.01m^2$）进行探测与跟踪。根据系统总体提出的技术指标， MTI/MTD 模块面临以下设计要求：

**频段（15.7-17.2GHz）**：雷达工作于 Ku 波段（波长 $\lambda \approx 1.8cm$），相比低频段雷达，同等速度目标产生的多普勒频移更大，地杂波频谱展宽也更为显著。MTI 滤波器必须具备深凹口抑制能力以应对展宽的杂波谱。同时，MTD 的多普勒滤波器组需要具备足够的频率分辨率，以区分密集的多普勒谱线。

**带宽（≥40MHz）**：距离分辨率为（$\Delta R \approx c/2B \approx 3.75\text{m}$），在 5km 的探测范围内，系统需要处理的距离门数量可达数千个，若考虑过采样则更多。MTI/MTD 模块的数据吞吐率要求极高。FPGA 设计必须采用流水线架构，处理时钟需匹配或倍频于采样率，且内部存储资源需支持数千个距离门的深度。

**数据吞吐率分析**： 承接前端脉冲压缩模块，本系统主数据流为 16-bit I/Q 复数信号。基于 60 MHz（**假设**） 的基带采样率，单通道数据带宽为：$60 \text{ Msps} \times 32 \text{ bit} = 1.92 \text{ Gbps} $。考虑到系统包含和、方位差、俯仰差及辅助四个同步通道，系统的总吞吐率需求达到 7.68 Gbps。为匹配高带宽需求，本设计采用 240 MHz 处理时钟结合时分复用，在单一物理通道上分时处理四路信号。

**盲速与盲距的解算**：系统脉冲周期范围为 $34\mu s - 125\mu s$，发射脉宽最大 $1\mu s$。单一高 PRF 虽然提高了不模糊测速范围，但会导致距离模糊；单一低 PRF 则会导致严重的速度模糊（盲速）。MTI/MTD 模块必须支持参差 PRF工作模式：采用组间参差策略，不同 CPI 使用不同的 PRF。模块内部参数需设计为可配置，能够随 PRF 动态切换，并通过后端算法解算真实速度和距离。

**高动态范围处理**：考虑到目标 RCS 动态范围大（$0.01m^2$ 至典型大目标），且需在强杂波背景下检测弱目标，信号处理全链路需保持高位宽精度。特别是在 MTI（最大增益 12dB）级联 MTD（相干积累增益约 20dB）的过程中，必须设计合理的动态截位策略，防止数据溢出导致虚假目标。
综上，MTI/MTD 系统需满足以下关键指标：

| 指标项            | 要求值                   | 备注                                 |
|:-------------- |:--------------------- |:---------------------------------- |
| **距离门数量**      | $\ge 2048$            | 覆盖 5 km 探测范围，距离分辨率 $\approx 3.75m$ |
| **PRF 支持范围**   | $8 kHz - 29kHz$       | 支持组间参差 PRF 以解速度模糊                  |
| **MTD FFT 点数** | 64 / 128 / 256 (可变)   | 根据驻留时间动态配置，典型值 128                 |
| **速度分辨率**      | $\le 0.5 m/s$         | 128点 Ku波段可达此精度                     |
| **杂波抑制能力**     | $> 40 dB$ (强杂波区)      | 3-Pulse MTI + MTD 增益               |
| **数据吞吐率**      | $\ge 7.68 Gbps$       | 满足 4 通道 $60MHz$ 实时处理               |
| **处理动态范围**     | $\ge 80 dB$           | 适配 $0.01m^2$ 至大目标 RCS 差异           |
| **通道一致性**      | 幅度$< 0.5dB$, 相位$< 2°$ | 保证“和差差”测角精度                        |
| **处理延迟**       | $< 1 CPI + 100\mu s$  | 实时性要求，主要延迟来自回波存储                   |

#### 设计方案分析

基于上述分析，制定如下核心技术方案：

**处理维度选择与储存架构**：采用脉冲维处理方案，即对同一距离单元在不同脉冲间的相位变化进行分析。采用 “存储-转置-流式处理”的处理流程。在脉冲压缩模块与 MTI 模块之间采用两个CPI内数据总量大小的乒乓结构。当一个 CPI 开始时，经过脉冲压缩的数据按快时间维即按列存入存储器中，存满一个 CPI 后再按慢时间即按行读出数据送入MTI/MTD模块，处理完成的数据再按行存回原来的位置，待一个 CPI 的数据全部处理完成后，再按列输出送入CFAR模块进行处理。

**自适应 MTI**：采用基于杂波图的自适应 MTI。利用旁路通道实时统计各距离门的零频能量，构建动态杂波图。主通道根据杂波图查找结果，实时在“旁路（Bypass）”、“二脉冲对消（2-Pulse）”和“三脉冲对消（3-Pulse）”三种模式间切换。

**MTD 相干积累**：采用加窗 FFT 滤波器组实现 MTD。通过对脉冲序列加窗抑制旁瓣，随后进行可变点 （64-512）FFT 运算，以匹配扫描跟踪模式。

### 总体结构

动目标检测整体结构如图所示：

![](assets/2026-01-30-16-30-26-MTD系统框图.png)

**数据分配模块**位于信号处理链路的最前端，该模块内部集成了两组RAM，构建起高效的“乒乓缓存”机制。在系统运行过程中，两组 RAM 交替进行读写操作：前端高速数据流按距离维顺序写入一组 RAM，同时后端处理引擎从另一组 RAM 中按脉冲维顺序读取上一帧数据。

**杂波感知模块**通过一条旁路通道，利用简化的双频点 DFT 算法快速提取每个距离单元回波的零频分量，以此表征该位置的静态地物强度。为了消除瞬时噪声的随机扰动并适应环境的缓慢变化，模块内部采用了递归加权算法对历史杂波图进行平滑更新。

**MTI 模块**是主信号通道的第一级处理器，本模块内建了可重构的 FIR 滤波引擎，能够根据杂波感知模块输出的控制信号在三种模式间动态切换：在杂波极弱的区域自动进入直通（Bypass）模式，完全保留低速目标能量；在中等杂波区启用二脉冲对消，兼顾抑制比与通带宽度；而在强杂波区则强制切换至三脉冲对消，利用其深凹口特性强力压制地物回波。

**MTD 模块**紧接 MTI 输出，通过将时域脉冲序列转换至多普勒频域，实现能量的相干积累与速度分辨。该模块内部集成了加窗、FFT 运算及动态截位三个子单元。数据首先经过加窗处理，随后进入可变点 FFT，最后的截位单元将运算后的高位宽数据规整为标准格式，以适配后续 CFAR 检测需求。

**控制模块**负责全系统的时序调度与资源管理，产生相应的使能信号对其他各个单元进行控制，保证整个电路的正确运行。

|           | `pc_last_i`   | 1   | Input  | **脉冲结束标志**。指示当前距离门序列的结束，用于控制存储器地址跳转。  |
|           | `pc_ch_id_i`  | 2   | Input  | **通道标识**。00:和, 01:方位差, 10:俯仰差, 11:辅助。 |
|           | `cpi_start_i` | 1   | Input  | **CPI同步信号**。指示一个新的相干处理间隔开始，触发乒乓缓存切换。  |
| **配置接口**  | `conf_data_i` | 32  | Input  | 配置数据（包含一个 CPI 内的脉冲数、脉冲采样数）。           |
|           | `conf_we_i`   | 1   | Input  | 配置写使能。                                |
| **数据输出**  | `mtd_data_o`  | 32  | Output | MTD处理后数据。                             |
|           | `mtd_valid_o` | 1   | Output | 输出数据有效指示。                             |
|           | `mtd_last_o`  | 1   | Output | 当前CPI数据传输结束标志。                        |
|           | `range_idx_o` | 13  | Output | 当前数据对应的**距离门索引**。                     |
|           | `dopp_idx_o`  | 9   | Output | 当前数据对应的**多普勒通道索引**。                   |
|           | `out_ch_id_o` | 2   | Output | 输出通道标识，与输入对应。                         |
